# GitHub Pages

This folder is ready for GitHub Pages as a simple static site.

## Files

- `index.html` - the page that shows the PDF in the browser
- `Eitan Dor.pdf` - the document itself
- `.nojekyll` - tells GitHub Pages to serve the site as-is

## Publish

1. Create a new GitHub repository.
2. Upload all files from this folder to the repository root.
3. In GitHub, open `Settings` -> `Pages`.
4. Under `Build and deployment`, choose `Deploy from a branch`.
5. Choose branch `main` and folder `/ (root)`.
6. Save and wait about 1-3 minutes.
7. Open the site at:

`https://YOUR-USERNAME.github.io/YOUR-REPO-NAME/`

## Notes

- The PDF stays on GitHub's hosting, not on your computer.
- If you replace the PDF later, keep the same file name or update `index.html`.
